# 변명아카이브 프론트엔드 작업 계획서

## 1. 프로젝트 개요

"칭찬 금지 기록 서비스" - 반성/후회 중심의 개인 기록 서비스
- 웹(React + Vite + TypeScript)
- 앱(React Native + Expo + TypeScript)

---

## 2. 기술 스택

### Web
- React 18+
- TypeScript
- Vite
- React Router
- TanStack Query (React Query)
- Zustand (상태관리)
- Tailwind CSS

### App (Mobile)
- React Native
- Expo + Expo Router
- TypeScript
- TanStack Query
- Zustand
- NativeWind (Tailwind for RN)

### 공통
- axios (API 클라이언트)
- zod (유효성 검증)

---

## 3. 작업 범위

### 3.1 공통 레이어 (packages/shared)
- [ ] API 클라이언트 모듈 (`apiClient.ts`)
- [ ] 요청/응답 타입 정의 (`types/`)
- [ ] 공통 에러 처리 유틸 (401/404/422/500)
- [ ] 유효성 검증 스키마 (zod)

### 3.2 인증
- [ ] 익명 시작 플로우 (`POST /api/auth/anonymous`)
- [ ] 토큰 저장/복원
  - 웹: localStorage
  - 앱: expo-secure-store
- [ ] 토큰 만료 시 재발급 처리

### 3.3 화면 구현

#### 홈 화면
- [ ] 최근 기록 3개 표시
- [ ] 오늘 한 줄 입력 (빠른 기록)
- [ ] 기록 작성 버튼

#### 기록 작성 화면
- [ ] 필수 입력 필드
  - occurredAt (발생 일시)
  - situation (상황)
  - myAction (내 행동)
  - result (결과)
  - cause (원인)
  - nextAction (다음 행동)
- [ ] 선택 입력 필드
  - recurrenceTrigger (재발 트리거)
  - recurrenceAction (재발 방지 행동)
  - mistakeType (실수 유형)
  - emotion (감정)
  - intensityLevel (강도 1-5)
- [ ] 클라이언트 선검증
- [ ] 칭찬 키워드 차단 시 422 안내

#### 기록 상세 화면
- [ ] 기록 전체 내용 표시
- [ ] nextActionDone 토글 (완료/미완료)
- [ ] 수정 기능
- [ ] 삭제 기능 (확인 모달)

#### 기록 목록 화면
- [ ] 페이지네이션
- [ ] 기록 카드 리스트
- [ ] 상세 페이지 이동

#### 패턴 리포트 화면
- [ ] 7일/30일 탭 전환
- [ ] Top3 실수 유형 시각화
- [ ] Top3 감정 시각화
- [ ] Top3 트리거 시각화

### 3.4 오류 UX
- [ ] 401: 토큰 재발급 유도 / 재시도
- [ ] 422: 칭찬 금지 안내 문구 표시
- [ ] 404: "기록을 찾을 수 없습니다"
- [ ] 500: 공통 오류 화면

## 4. API 엔드포인트

| Method | Endpoint | 설명 |
|--------|----------|------|
| POST | `/api/auth/anonymous` | 익명 토큰 발급 |
| POST | `/api/records` | 기록 생성 |
| GET | `/api/records` | 기록 목록 조회 |
| GET | `/api/records/{id}` | 기록 상세 조회 |
| PATCH | `/api/records/{id}` | 기록 수정 |
| DELETE | `/api/records/{id}` | 기록 삭제 |
| GET | `/api/reports/patterns?days=7\|30` | 패턴 리포트 |

## 5. 도메인 필드

```typescript
interface Record {
  id: number;
  occurredAt: string; // ISO8601
  situation: string;
  myAction: string;
  result: string;
  cause: string;
  nextAction: string;
  nextActionDone: boolean;
  recurrenceTrigger?: string;
  recurrenceAction?: string;
  mistakeType?: MistakeType;
  emotion?: Emotion;
  intensityLevel?: number; // 1-5
  createdAt: string;
  updatedAt: string;
}

type MistakeType =
  | 'PROCRASTINATION'
  | 'TIME_MANAGEMENT'
  | 'COMMUNICATION'
  | 'FOCUS'
  | 'EMOTIONAL'
  | 'JUDGMENT'
  | 'AVOIDANCE'
  | 'OTHER';

type Emotion =
  | 'ANXIETY'
  | 'GUILT'
  | 'EMBARRASSMENT'
  | 'RESTLESSNESS'
  | 'HELPLESSNESS'
  | 'ANGER'
  | 'SADNESS'
  | 'OTHER';
```

## 7. 완료 조건 (DoD)

1. 사용자 시나리오가 웹/앱 모두 동작
   - 최초 진입 → 익명 토큰 발급 → 기록 생성 → 목록/상세 확인 → 리포트 조회

2. API 오류 상태별 UI 분리 처리
   - 401, 422, 404, 500

3. 웹/앱에서 동일한 도메인 필드 구조 사용

4. QA 체크리스트 통과
   - 저장 성공/실패
   - 수정/삭제
   - 패턴리포트 탭 전환

---

## 8. 검증 시나리오

1. 익명 토큰 발급 후 기록 생성
2. 기록 수정 (nextActionDone true/false)
3. 칭찬 키워드 기록 저장 시 422 메시지 노출
4. 패턴리포트 7/30일 데이터 렌더링
