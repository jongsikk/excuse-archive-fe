# Design System: Reflective Editorial

## 1. Overview & Creative North Star: "The Curated Gallery"
This design system moves away from the aggressive, high-contrast patterns of typical utility apps and leans into the world of high-end digital editorial. Our Creative North Star is **The Curated Gallery**: an experience that feels quiet, intentional, and expensive.

We achieve this through "Breathable Asymmetry." Instead of forcing content into a rigid, boxed grid, we use expansive white space and intentional overlaps to guide the eye. This system is designed to reduce cognitive load and visual fatigue, replacing "neon" energy with "sage" wisdom. Every element should feel like it was placed by a human curator, not generated by a template.

---

## 2. Colors: Tonal Depth & The Sage Palette
The palette is rooted in muted organic tones. We avoid pure blacks and harsh whites to mimic the soft diffusion of natural light on premium matte paper.

### Core Palette
*   **Primary (`#2e6863`)**: A sophisticated, muted Sage. Use this for moments of quiet authority.
*   **Secondary (`#4a6461`)**: A weathered Slate-Mint, used for supporting elements.
*   **Neutral / On-Surface (`#2c333a`)**: A soft Charcoal. Never use pure black (#000); this charcoal maintains a soft, "ink-on-paper" legibility.

### The "No-Line" Rule
**Borders are prohibited for sectioning.** To create separation, use the "Surface Shift" method.
*   **Transitioning Layers:** Define a section boundary by moving from `surface` (#f7f9fe) to `surface-container-low` (#f0f4fa).
*   **Interaction:** If an element needs to feel interactive, use a background color shift rather than an outline.

### The "Glass & Gradient" Rule
To add soul to the interface, avoid flat blocks of color. 
*   **Signature Textures:** Use a linear gradient for large CTAs or Hero backgrounds transitioning from `primary` (#2e6863) to `primary_container` (#b3eee7) at a 120-degree angle with 40% opacity.
*   **Glassmorphism:** For floating navigation or modal overlays, use `surface_container_lowest` (#ffffff) with an 80% opacity and a `20px` backdrop-blur.

## 3. Typography: The Editorial Voice
We use **Manrope** exclusively. Its geometric yet humanist qualities allow it to function as both a high-fashion display face and a highly legible functional typeface.

*   **Display (lg/md/sm):** Use `display-lg` (3.5rem) for hero moments. Apply a slightly tighter letter-spacing (-0.02em) to create a "locked" editorial feel.
*   **Headlines:** Use `headline-md` (1.75rem) in `on_surface` charcoal. Ensure these have significant top-margin (Spacing 12 or 16) to let themes breathe.
*   **Body (lg/md):** All body copy defaults to `on_surface_variant` (#586068) to keep the "muted" promise. Only use `on_surface` (#2c333a) for emphasized text.
*   **Labels:** Use `label-md` (0.75rem) in all-caps with +0.05em tracking for a "metadata" or "archival" aesthetic.

## 4. Elevation & Depth: Tonal Layering
In this system, depth is a physical property, not a visual decoration. We use **Tonal Layering** instead of structural lines.

### The Layering Principle
Think of the UI as a stack of fine cardstock.
1.  **Base:** `surface` (#f7f9fe)
2.  **Sectioning:** `surface_container_low` (#f0f4fa)
3.  **Interactive Cards:** `surface_container_lowest` (#ffffff) placed on top of the sectioning layer. This creates a natural "lift" without a single pixel of shadow.

### Ambient Shadows
When an element must float (e.g., a primary action card), use an **Ambient Shadow**:
*   **Color:** `on_surface` (#2c333a) at 6% opacity.
*   **Blur:** 24px to 40px.
*   **Offset:** Y: 8px.
*   *Note: Never use a shadow color that defaults to pure black; it must be tinted with our charcoal.*

### The "Ghost Border" Fallback
If accessibility requirements demand a border (e.g., in high-contrast modes), use `outline_variant` (#abb3bc) at **15% opacity**. It should be felt, not seen.

## 5. Components: Soft Interaction

### Buttons
*   **Primary:** `primary` background with `on_primary` text. Use `xl` (0.75rem) corner radius. No shadows.
*   **Secondary:** `surface_container_high` background. This should blend almost entirely with the background until hovered.
*   **Tertiary:** Text-only using `primary` color with a subtle `3px` underline in `primary_container`.

### Cards & Lists
*   **The No-Divider Rule:** Forbid horizontal lines between list items. Use `Spacing 4` (1.4rem) of vertical whitespace or a alternating background of `surface` and `surface_container_low`.
*   **Cards:** Use `xl` (0.75rem) or `lg` (0.5rem) rounding. Avoid borders; use a soft Ambient Shadow (6% opacity) for "active" cards.

### Input Fields
*   **Default State:** `surface_container_highest` background with a `Ghost Border`.
*   **Focus State:** Background shifts to `surface_container_lowest` (pure white) with a 2px `primary` bottom-border only. This mimics a signature line on a document.

### Archive Chips
*   Used for categorization. Style with `secondary_container` (#cce8e4) and `on_secondary_container` (#3d5654). No border, `full` (9999px) rounding.

---

## 6. Do’s and Don’ts

### Do
*   **Do** use asymmetrical margins. For example, give a headline a larger left-margin than the body text to create an editorial "indent."
*   **Do** lean into `surface_container_lowest` (pure white) for the most important content to make it "pop" against the off-white background.
*   **Do** use the Sage `primary` color sparingly. It is a highlighter, not a bucket-fill.

### Don’t
*   **Don't** use 1px solid borders to separate content.
*   **Don't** use pure black (#000000) for text; it breaks the "Reflective" atmosphere.
*   **Don't** use "standard" 8px or 16px padding everywhere. Mix the spacing scale (e.g., use `Spacing 10` for top padding and `Spacing 6` for sides) to create an intentional, non-templated look.
*   **Don't** use high-contrast, fast animations. Use `cubic-bezier(0.2, 0, 0, 1)` for all transitions to ensure they feel fluid and "weighted."
