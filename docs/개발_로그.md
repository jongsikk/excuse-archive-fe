## [2026-03-31] 인사이트 기능 1차 개발

### 개요
사용자가 기록을 **습관화**하고 기록에서 **실제 인사이트**를 얻도록 하는 기능 4종을 앱/웹 동시 구현.

### 구현 기능

#### 1. 연속 기록 스트릭
- 홈 화면에 🔥 N일 연속 기록 배너 표시
- 오늘 또는 어제 기록이 있으면 연속일 계산 시작
- 기록 없는 날 스트릭 초기화

#### 2. 미완료 nextAction 배너
- 홈 화면에 📌 "아직 못 한 것들 N개" 배너 표시
- 탭/클릭하면 기록 목록의 미완료 탭으로 이동

#### 3. 주간 요약 카드
- 홈 화면에 지난 7일 총 기록 / 완료율 / 가장 많이 반복된 실수 유형 표시

#### 4. 미완료 필터 탭 (기록 목록)
- 전체 / 미완료 탭 전환
- 미완료 없을 때 🎉 빈 상태 메시지

#### 5. 감정 강도 트렌드 차트 (리포트)
- 일별 평균 강도를 바 차트로 시각화
- 낮음(민트) / 보통(노랑) / 높음(빨강) 색상 구분
- 7일: 전체 표시, 30일: 5일 간격으로 표시

### 변경된 파일

| 파일 | 변경 내용 |
|------|-----------|
| `packages/shared/src/utils/insights.ts` | 새로 생성 — calculateStreak, calculateWeeklySummary, calculateIntensityTrend |
| `packages/shared/src/index.ts` | utils/insights 익스포트 추가 |
| `apps/mobile/lib/insights.ts` | shared 패키지 re-export로 변경 |
| `apps/mobile/app/(tabs)/index.tsx` | 스트릭 + 미완료 배너 + 주간 요약 추가 |
| `apps/mobile/app/(tabs)/records.tsx` | 전체/미완료 필터 탭 추가 |
| `apps/mobile/app/(tabs)/report.tsx` | 감정 강도 트렌드 차트 추가 |
| `apps/web/src/pages/HomePage.tsx` | 스트릭 + 미완료 배너 + 주간 요약 추가 |
| `apps/web/src/pages/RecordListPage.tsx` | 전체/미완료 필터 탭 + URL 파라미터 연동 |
| `apps/web/src/pages/ReportPage.tsx` | 감정 강도 트렌드 차트 추가 |

### 기술 결정
- 인사이트 유틸 함수를 `packages/shared`에 배치 → 모바일/웹 코드 중복 없음
- 외부 차트 라이브러리 없이 직접 구현 → 의존성 최소화
- 스트릭 계산: UTC 기준 날짜 비교 (서버 저장 포맷과 일치)
- 미완료 필터: 최대 200개 fetch 후 클라이언트 필터링 (별도 API 불필요)

## [2026-03-31] 환경 설정 및 디버깅

### 해결한 이슈

| 문제 | 원인 | 해결 |
|------|------|------|
| `simctl` 명령 실패 | xcode-select가 CommandLineTools 가리킴 | `sudo xcode-select -s /Applications/Xcode.app/Contents/Developer` |
| `.plugins is not a valid Plugin property` | NativeWind v4에서 babel plugin 방식 제거됨 | `babel.config.js`에서 `plugins: ['nativewind/babel']` 삭제 |
| `favicon.png not found` | assets 폴더 비어 있음 | 1x1 PNG 파일 4종 생성 |
| `expo-linking` 모듈 없음 | 모노레포 호이스팅 문제 | 루트에서 `npm install` 재실행 |
| `react-native-css-interop` 없음 | nativewind 내부 중첩 설치됨 | 루트 레벨로 재설치 |
| NativeWind 스타일 미적용 | `metro.config.js` 없음 | `withNativeWind` 래퍼로 metro 설정 생성 |
| `occurredAt` 400 에러 | 서버가 `java.time.Instant` 기대, timezone 없는 포맷 거부 | ISO 8601 UTC 포맷(`Z` suffix) 사용 |

전체 7개 엔드포인트 정상 동작 확인 (상세: `docs/API_테스트_결과.md`)

## [2026-03-31] 디자인 개선 2차 — 피그마 기반 UI 발전

### 개요
Figma 파일 분석 후 미구현 기능 4가지를 앱/웹 동시 구현.

### 구현 기능

#### 1. 기록 히트맵 (홈 화면)
- 최근 12주(웹) / 8주(앱) 기록을 달력 형태로 시각화
- 강도 기반 5색 색상: 없음(회색) → 낮음(초록) → 보통(노랑) → 높음(빨강)
- 오늘 날짜 민트 테두리 표시
- 강도 없이 기록만 있으면 민트색으로 표시

#### 2. 스텝형 기록 작성 폼
- 3단계 UI: Step1(무슨 일이?) → Step2(왜 그랬지?) → Step3(다음엔?)
- 단계별 진행 표시 바 (완료 체크, 컬러 구분)
- Step1: 상황/내행동/결과, Step2: 원인/실수유형/감정/강도, Step3: 다음행동/재발방지
- 단계별 유효성 검사 (필수 필드 통과 후 다음 단계)

#### 3. 리포트 라인 차트 오버레이 (웹)
- 기존 바 차트에 민트색 SVG 라인 오버레이 추가
- 바 차트(반투명) + 라인 + 원형 포인트 조합
- 시각적 트렌드 파악 용이

#### 4. 강도 선택 UI 개선 (모바일 폼)
- 슬라이더 → 1~5 원형 버튼으로 변경
- 강도별 색상 피드백 (초록/노랑/주황/빨강)

### API 스펙

```
POST /api/auth/register
Body: { email, password, displayName? }
→ 200: { accessToken, externalId }
→ 400: { error: "이미 사용 중인 이메일입니다." }

POST /api/auth/login
Body: { email, password }
→ 200: { accessToken, externalId }
→ 401: { error: "이메일 또는 비밀번호가 올바르지 않습니다." }
```
